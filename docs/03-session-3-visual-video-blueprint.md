# Session 3 — Visual / Video Blueprint

## Project
Nano Line Healthy Bread Production Line  
Model: **NL-RM210 / Steel Master**

---

# 0. Visual Direction Summary

## Overall Visual Style

The landing page should feel like a premium industrial B2B website, not a PDF brochure.

The visual style should be:

- Clean
- Industrial
- Modern
- Technical
- Trust-building
- Factory-focused
- Rich with visuals, but not noisy
- White / light background with steel, navy, and controlled accent colors

## Main Visual Concept

The website should explain the production line visually, from business problem to technical solution, then from dough input to finished bread output.

The user should understand:

1. This is a custom production line.
2. The bread format can be adjusted.
3. The line is scalable.
4. The process is complete.
5. Pricing requires technical review.
6. The sales team needs project data before quotation.

# 3. Visual Asset Quality Rules

## Do Not

- Do not use raw PDF screenshots directly as final website visuals.
- Do not show PDF page numbers, quotation numbers, or pricing tables.
- Do not use low-resolution cropped machine images without cleaning.
- Do not show any dollar signs or price-related visuals.
- Do not make the website look like an uploaded brochure.
- Do not overpromise fixed capacity without stating that configuration affects output.

## Do

- Use PDF visuals as reference material.
- Recreate weak visuals as AI-enhanced or 3D-style industrial visuals.
- Keep the factory environment clean and realistic.
- Show food safety through uniforms, clean floors, stainless steel, controlled lighting, and organized production flow.
- Use diagrams and motion graphics to explain complex technical points.
- Use image captions to educate the client.

# 8. Implementation Notes for Web Build

## Desktop Experience

- Use a strong hero image or loop video above the fold.
- Add sticky CTA after user scrolls past the hero.
- Use scroll-triggered production stages.
- Place calculator before the long technical stage section.
- Use visual cards instead of long paragraphs where possible.
- Keep the form visually premium and not too long on one screen.

## Mobile Experience

- Hero video should be lightweight or replaced with image.
- Use accordion format for stages.
- Calculator should be simple and thumb-friendly.
- Sticky WhatsApp / quotation CTA should appear after first scroll.
- Avoid heavy auto-playing videos on mobile.

## Performance Rules

- Compress all images.
- Use WebP or AVIF for images.
- Use MP4/WebM for loops.
- Lazy-load below-the-fold visuals.
- Avoid loading all stage videos at once.
- Use poster images for videos.

## Accessibility Notes

- Every technical image should have descriptive alt text.
- Do not rely only on visual icons; include short labels.
- Calculator results should be readable as text.
- Form errors should be clear and human-friendly.

# 10. Final Creative Direction

The page should feel like the client is entering a technical sales consultation, not browsing a catalog.

Every visual should answer a sales question:

- What does the line look like?
- What stages does it include?
- Can it produce my bread shape?
- Can it meet my capacity?
- What add-ons can I request?
- What space do I need?
- Why do I need a custom quotation?

The best visual direction is a mix of:

- Real machine references from the Technical Offer
- AI-enhanced factory visuals
- Clean technical diagrams
- Interactive calculator UI
- Motion graphics for process explanation

The final result should be a visually rich B2B landing page that educates the client and gives the sales team a qualified lead.
